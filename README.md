# Квадрат задач

PWA-приложение для личного управления задачами: быстрый ввод, «Сегодня / Завтра / Неделя», матрица Эйзенхауэра, ABCDE, 1–3–5, Канбан, проекты, архив и табель занятости по проектам.

## Что есть в версии 2.0

- быстрый ввод задачи одной строкой;
- расширенное создание: проект, срок, дата плана, статус, приоритет, важность, срочность, комментарий;
- главный экран «Сегодня» по логике Айви Ли + 1–3–5;

- быстрые теги проектов (#МЗМО, #РДКБ, #Сколтех и любые свои), которые автоматически подставляют проект в новую задачу и при редактировании;
- исправлен контраст текста в выпадающих списках;
- улучшена верстка карточек и Канбана: длинные ссылки и комментарии больше не ломают карточку;
- Канбан стал компактным: в колонках выводятся проекты, а задачи внутри проекта показаны короткими ссылками для открытия карточки;
- табель теперь можно смотреть по всем проектам или по выбранному проекту отдельно;
- добавлен свод и разделение часов по проектам в табеле.
- экраны «Завтра», «Неделя», «Разбор», «Эйзенхауэр», «Канбан», «Проекты», «Архив»;
- новая вкладка «Табель»;
- отметка «работал над проектом» из карточки проекта или из задачи;
- журнал занятости: дата, проект, часы, код явки/неявки, комментарий;
- автоматическая сборка табеля с полями: ФИО, должность, дни 1–15, итог за 1–15, дни 16–31, итог за месяц;
- экспорт табеля в Excel-совместимый `.xls` без сторонних библиотек;
- экспорт журнала отметок в CSV;
- локальная память в браузере;
- экспорт и импорт JSON;
- PWA-режим для установки на iPhone и ноутбук;
- опциональная синхронизация через облако Auth + Postgres.

## Как отмечать работу по проекту

1. Открой вкладку «Проекты».
2. Нажми «Отметить сегодня» в нужном проекте.
3. По умолчанию будет записано 8 часов с кодом «Я».

Альтернатива: в задаче с указанным проектом нажми «Работал».

Во вкладке «Табель» можно вручную добавить отметку: дата, проект, часы, код, комментарий.

Коды:

- `Я` — явка;
- `В` — выходной;
- `Б` — больничный;
- `ОТ` — отпуск;
- `НН` — неявка.

## Экспорт табеля

1. Открой вкладку «Табель».
2. Выбери месяц.
3. Нажми «Экспорт табеля Excel».
4. Скачается файл `.xls`, который открывается в Excel.

Формат сделан по логике загруженного примера: строка часов по дням, строка кодов явки/неявки, итог за 1–15 и итог за месяц.

## Размещение на GitHub Pages

2. Загрузи в него все файлы из этой папки.
3. Открой Settings → Pages.

5. Branch: main, folder: /root.
6. Сохрани и открой выданную GitHub Pages ссылку.

2. Нажми «Поделиться».
3. Выбери «На экран Домой».
4. Открой приложение с иконки.

## Синхронизация через облако

Приложение работает без облако. Но синхронизация между iPhone и ноутбуком требует облачной базы.

1. Создай бесплатный проект на облако.
2. В облако открой SQL Editor.
3. Вставь SQL из файла `cloud schema`.
4. В облако открой Project Settings → API.
5. Скопируй Project URL и anon public key.
6. В приложении открой «Синхронизация» и вставь эти значения.
7. Укажи email и отправь magic link.
8. После входа нажми «Синхронизировать».

## Важное ограничение

GitHub Pages хранит только код приложения. Локальные задачи и табель хранятся в браузере устройства. Для настоящей синхронизации нужен облако или другой backend.


## Исправление в версии 1.4

Исправлена ошибка синхронизации облако: `value is out of range for type integer`.

Причина: приложение использует `Date.now()` для внутреннего порядка задач, а это число больше стандартного PostgreSQL `integer`. В таблице `tasks` поле `order_index` теперь должно быть `bigint`.

Для уже созданной базы один раз выполните в облако → SQL Editor:

```sql
alter table public.tasks alter column order_index type bigint using order_index::bigint;
```

## Версия 2.10.4 — Conflict-Safe Sync

Исправление проблемы «синхронизируется один раз» и возврата удалённых задач.

Причина:
- кнопка «Синхронизировать» в v2.10.3 отправляла весь локальный архив перед чтением облака;
- второе устройство могло перезаписать удаление старой активной локальной копией задачи;
- из-за этого удалённые задачи возвращались, а дальнейшая синхронизация выглядела как сломанная.

Новая логика одной кнопки:
1. Сначала прочитать задачи из Supabase.
2. Слить облако с локальными изменениями, не затирая новые/удалённые локальные задачи.
3. Отправить только локальные изменения из очереди dirty-задач.
4. Ещё раз прочитать Supabase и обновить экран.

Важно:
- обычная кнопка больше не выгружает весь локальный архив;
- удаление на одном устройстве не должно перетираться старой копией со второго устройства;
- интерфейс остаётся простым: одна кнопка «Синхронизировать».

## Версия 2.11.0 — Cloud-First Tasks

Перестроена логика задач: больше не пытаемся синхронизировать два локальных архива как равноправные источники.

Новая модель:
- при активном входе создание задачи сразу пишет строку в Supabase;
- изменение задачи сразу пишет строку в Supabase;
- удаление задачи сразу пишет `deleted_at` в Supabase;
- кнопка «Синхронизировать» на втором устройстве в первую очередь забирает актуальное состояние из Supabase;
- локальная очередь остаётся только как аварийный fallback, если интернет или облако недоступны.

Почему это сделано:
- предыдущая модель могла один раз успешно передать задачу, а затем конфликтовать при удалении и повторной синхронизации;
- устройство со старой локальной копией могло вернуть задачу, которую уже удалили на другом устройстве.

Пользовательский сценарий:
1. На iPhone создать задачу.
2. Она сразу сохраняется в облако.
3. На компьютере нажать «Синхронизировать».
4. Задача появляется.
5. Удалить задачу на компьютере.
6. Удаление сразу сохраняется в облако.
7. На iPhone нажать «Синхронизировать».
8. Задача исчезает.

## Версия 2.12.0 — Sync Engine v1

Реализован основной контур полуавтоматической синхронизации обычных задач на Supabase.

Принцип:
- Supabase — источник правды по задачам;
- localStorage — локальный кэш;
- создание/изменение/удаление задачи сразу пытается сохраниться в Supabase;
- если облако недоступно, задача остаётся в очереди `dirtyTaskIds`;
- кнопка «Синхронизировать» отправляет ожидающие изменения, читает Supabase по `updated_at desc`, обновляет локальный кэш и скрывает удалённые задачи;
- удаление выполняется через `deleted_at`, физическое удаление строк не используется.

Не менялось: интерфейс приложения, проекты, канбан, табель, профиль, админка, Sync Lab / Read Inspector.


## Версия 2.12.1 — Stable Cloud Sync

Стабилизация синхронизации после проверки v2.12.0.

Главные изменения:
- кнопка «Синхронизировать» выполняет полный стабильный цикл: отправка ожидающих → полное чтение Supabase → пересборка localStorage;
- чтение задач выполняется постранично через `.range()`;
- localStorage при синхронизации пересобирается от Supabase как от главного источника;
- dirty-задачи не затираются облачной копией, если они ещё не подтверждены сервером;
- dirty-флаг снимается только после явного ответа Supabase через `.select('id,updated_at,deleted_at,title')`;
- auto-pull на `focus`, `visibilitychange`, `online` отключён, чтобы убрать гонки;
- создание, изменение и удаление задач продолжают сразу пытаться сохраниться в Supabase;
- удаление выполняется через `deleted_at`.

Тест:
1. Создать 3 обычные задачи на iPhone.
2. Убедиться, что все 3 появились в Supabase.
3. На компьютере нажать «Синхронизировать».
4. Все 3 должны появиться.
5. Удалить 1 задачу на компьютере.
6. На iPhone нажать «Синхронизировать».
7. Удалённая задача должна исчезнуть и не вернуться.


## Версия 2.12.2 — Tombstone Wins

Исправление возврата удалённых задач.

Правило:
- если Supabase вернул задачу с заполненным `deleted_at`, это окончательное удалённое состояние;
- локальная копия, dirty-флаг или более свежий локальный `updatedAt` не могут воскресить такую задачу;
- dirty-флаг удалённой облачной задачи очищается;
- `activeTasks()`, `visibleTasks()`, `listHtml()` и `taskCard()` получили дополнительную защиту: удалённая задача не может попасть в рабочий список даже при ошибочном вызове.

Проверка:
1. На iPhone удалить задачу.
2. Убедиться в Supabase, что `deleted_at` заполнен.
3. На компьютере нажать «Синхронизировать».
4. Задача не должна появиться в «Разборе».
5. Повторить в обратную сторону.


## Версия 2.12.3 — Cloud Pull Wins

Исправление ситуации, когда задача создана на iPhone, есть в Supabase, но не появляется на компьютере.

Изменения:
- перед отправкой локальной dirty-задачи приложение проверяет эту же строку в Supabase;
- если в Supabase у задачи уже есть `deleted_at`, локальная копия не отправляется и не может воскреснуть;
- после ручной синхронизации список задач пересобирается от Supabase как от главного источника;
- локальные dirty-задачи сохраняются только если Supabase ещё не знает эту строку;
- после ручной синхронизации сбрасываются фильтры поиска/проекта;
- при наличии входящих задач после синхронизации приложение переводится на экран «Разбор»;
- отключены оставшиеся фоновые авто-движки `enqueueAutoTaskSync`, `enqueueAutoPull`, `scheduleReliableRetry`, чтобы они не мешали ручному cloud-pull.

Проверка:
1. Создать задачу на iPhone.
2. Убедиться, что она появилась в Supabase и `deleted_at = null`.
3. На компьютере нажать «Синхронизировать».
4. Компьютер должен перейти в «Разбор», задача должна быть видна.
