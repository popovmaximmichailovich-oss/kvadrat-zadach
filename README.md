# Квадрат задач

PWA-приложение для личного управления задачами: быстрый ввод, «Сегодня / Завтра / Неделя», матрица Эйзенхауэра, ABCDE, 1–3–5, Канбан, проекты, архив и табель занятости по проектам.

## Что есть в версии 1.4

- быстрый ввод задачи одной строкой;
- расширенное создание: проект, срок, дата плана, статус, приоритет, важность, срочность, комментарий;
- главный экран «Сегодня» по логике Айви Ли + 1–3–5;

- быстрые теги проектов (#МЗМО, #РДКБ, #Сколтех и любые свои), которые автоматически подставляют проект в новую задачу и при редактировании;
- исправлен контраст текста в выпадающих списках;
- улучшена верстка карточек и Канбана: длинные ссылки и комментарии больше не ломают карточку;
- Канбан стал компактным: в колонках выводятся проекты, а задачи внутри проекта показаны короткими ссылками для открытия карточки;
- табель теперь можно смотреть по всем проектам или по выбранному проекту отдельно;
- добавлен свод и разделение часов по проектам в табеле.
- экраны «Завтра», «Неделя», «Разбор», «Эйзенхауэр», «Канбан», «Проекты», «Архив»;
- новая вкладка «Табель»;
- отметка «работал над проектом» из карточки проекта или из задачи;
- журнал занятости: дата, проект, часы, код явки/неявки, комментарий;
- автоматическая сборка табеля с полями: ФИО, должность, дни 1–15, итог за 1–15, дни 16–31, итог за месяц;
- экспорт табеля в Excel-совместимый `.xls` без сторонних библиотек;
- экспорт журнала отметок в CSV;
- локальная память в браузере;
- экспорт и импорт JSON;
- PWA-режим для установки на iPhone и ноутбук;
- опциональная синхронизация через Supabase Auth + Postgres.

## Как отмечать работу по проекту

1. Открой вкладку «Проекты».
2. Нажми «Отметить сегодня» в нужном проекте.
3. По умолчанию будет записано 8 часов с кодом «Я».

Альтернатива: в задаче с указанным проектом нажми «Работал».

Во вкладке «Табель» можно вручную добавить отметку: дата, проект, часы, код, комментарий.

Коды:

- `Я` — явка;
- `В` — выходной;
- `Б` — больничный;
- `ОТ` — отпуск;
- `НН` — неявка.

## Экспорт табеля

1. Открой вкладку «Табель».
2. Выбери месяц.
3. Нажми «Экспорт табеля Excel».
4. Скачается файл `.xls`, который открывается в Excel.

Формат сделан по логике загруженного примера: строка часов по дням, строка кодов явки/неявки, итог за 1–15 и итог за месяц.

## Размещение на GitHub Pages

1. Создай новый публичный репозиторий на GitHub.
2. Загрузи в него все файлы из этой папки.
3. Открой Settings → Pages.
4. Source: Deploy from a branch.
5. Branch: main, folder: /root.
6. Сохрани и открой выданную GitHub Pages ссылку.

## Установка на iPhone

1. Открой опубликованную ссылку в Safari.
2. Нажми «Поделиться».
3. Выбери «На экран Домой».
4. Открой приложение с иконки.

## Синхронизация через Supabase

Приложение работает без Supabase. Но синхронизация между iPhone и ноутбуком требует облачной базы.

1. Создай бесплатный проект на Supabase.
2. В Supabase открой SQL Editor.
3. Вставь SQL из файла `supabase.sql`.
4. В Supabase открой Project Settings → API.
5. Скопируй Project URL и anon public key.
6. В приложении открой «Синхронизация» и вставь эти значения.
7. Укажи email и отправь magic link.
8. После входа нажми «Синхронизировать».

## Важное ограничение

GitHub Pages хранит только код приложения. Локальные задачи и табель хранятся в браузере устройства. Для настоящей синхронизации нужен Supabase или другой backend.


## Исправление в версии 1.4

Исправлена ошибка синхронизации Supabase: `value is out of range for type integer`.

Причина: приложение использует `Date.now()` для внутреннего порядка задач, а это число больше стандартного PostgreSQL `integer`. В таблице `tasks` поле `order_index` теперь должно быть `bigint`.

Для уже созданной базы один раз выполните в Supabase → SQL Editor:

```sql
alter table public.tasks alter column order_index type bigint using order_index::bigint;
```
